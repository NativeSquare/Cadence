# Abandoning a Goal archives and prunes, rather than deletes

Changing your Goal is the most common cause of a real bug: the Calendar and the "To categorize" screen show **multiple workouts per day**, because the previous goal's workouts linger alongside the new plan's. The mechanism: `listWorkouts` (`agoge/workouts.ts`) is keyed by **athlete**, not by plan, so it returns every workout the athlete ever had. And the old "change goal" path *deleted the Race*, which cascaded through Agoge's `cascadeDeletePlan` — a cascade that does **not** delete the plan's workouts, it **detaches** them (`planId/blockId → undefined`). So each goal change left a pile of orphaned, plan-less prescriptions that the athlete-keyed read faithfully resurfaced forever. Worse, an orphaned workout and an *intentionally* plan-less "free workout" (created off a plan's date window, on purpose) are then indistinguishable — both `planId: undefined` — so a naive "filter the read to the active plan" fix would also hide every free workout the runner deliberately scheduled.

We make goal abandonment a **lossless archive plus a targeted prune**, fixing the problem at the **source** (the write) rather than the read. On "change goal", for the race Goal (the only kind — see below):

- the **Race** moves to `status: "cancelled"` — which (a) auto-derives the **Goal** to `abandoned` via `deriveGoalStatus`, the model's existing "goal status follows race status" invariant, and (b) clears the `no-upcoming-A-race-conflict` check, the sole reason the old code deleted the race;
- the **Plan** is archived (`archivedAt`, via the existing `archivePlansForRace` helper) — never deleted;
- the plan's Workouts are split by a single test — **presence of `actual`**: those *with* recorded actuals (real training that happened) are **kept**, attached to the now-archived plan as durable history; those *without* (planned, needs-feedback, and runner-marked `missed` alike) are **deleted** as disposable, deterministically regenerable prescriptions.

We also **remove the general-fitness Goal category outright** (`createMyFitnessGoal`, `fitnessIntentValidator`, and the fitness branches in `getMyActiveGoal`/`getAthletePlan`). It was already declared "dropped for MVP" in `CONTEXT.md` but had drifted back into live code; with it gone, every Goal is race-anchored and the abandon path is single-shape. No production data exists for it, so this is a clean greenfield removal — no migration.

## Consequences

- **`listWorkouts` needs no change, and no per-workout "archived" flag is introduced.** Once the dead prescriptions are gone at the source, the athlete-keyed read naturally returns exactly the right set: active-plan workouts + the archived plan's *completed* history + any intentionally-free workouts. We deliberately avoided a soft-delete/archived-status dimension on workouts — that would be a status machine every read path (`getPlannedWorkoutsByAthlete`, analytics, Guardrail counts) must remember to honour, and forgetting it would resurrect the phantom. Deleting the prescriptions keeps reads dumb.
- **The keep/delete test is `actual` presence, not the `status` string.** "Did real training happen?" is the whole rule. This is robust to status-field edge cases and matches the value asymmetry: a done Workout is irreplaceable (actuals, VDOT basis, journal links); a planned one is regenerable from goal + VDOT.
- **No schema change and no Agoge-library change.** The whole lifecycle is expressed with states the model already has (`race.status: "cancelled"`, `plan.archivedAt`, derived goal status) and lives entirely in the host-app abandon mutation. Agoge's detach-on-`cascadeDeletePlan` behaviour is simply no longer on the path, because we archive the plan instead of deleting it.
- **Abandonment is now consistent with the "goal status is derived" invariant.** The previous `abandonMyActiveGoal` patched `goal.status` directly to `abandoned` for fitness goals — off-pattern. Routing through `race.status` is the one true way goal status changes.
- **Cancelled races persist** as provenance anchors and will appear in athlete-wide race reads (`listMyRaces*`). If old cancelled races later become their own clutter in a races list, that is a separate read-side display decision — out of scope here.
- **Pre-existing orphans are not addressed by this change.** Workouts already detached by the old delete-the-race path remain `planId: undefined` and indistinguishable from intentional free workouts; cleaning them is a one-off the owner handles manually, not part of this contract.
