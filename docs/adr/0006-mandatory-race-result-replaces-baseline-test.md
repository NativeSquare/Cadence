# A mandatory past race result replaces the in-app baseline test

Plan generation needs a VDOT. We previously seeded it two ways: an optional self-reported recent race, or — if skipped — a scheduled in-app "5K time trial" **test** workout the runner completed to derive VDOT. We **remove the test entirely** and make a **past race result mandatory** at onboarding: a standard distance (`5k`/`10k`/`half_marathon`/`marathon` — the same set as the goal race; 15K is dropped so the two are unified) plus a finish time. The input stays the existing structured distance-pill + h/m/s time field — never free-form. The race result is **folded into `createMyRaceWithGoal`**, which writes the VDOT metric and then generates the plan in one transaction, making "VDOT exists before generation" a **server-side invariant** rather than a client call-ordering convention (the old order — create-goal-then-set-VDOT — is precisely why the placeholder test workout had to exist).

This is a deliberate positioning choice, not a technical gate: "official race only" is enforced by copy (officialness can't be verified), and it narrows the target to runners with a real race history.

## Consequences

- **`gatePlanGeneration` collapses to scheduling `generate`.** The no-VDOT branch, the test workout (`buildTestStructure`, `TEST_NAME`/`TEST_DESCRIPTION`), `recordVdotFromCompletedTest`, the "delete pending test" loop, the test result-entry UI (`mark-done-bottom-sheet`'s `isTest` form, `workout-detail-page`'s `isBaselineTest`, `NeedsFeedbackScreen`'s `isTest`), and `capturePostSession`'s `testDistanceMeters`/`testDurationSeconds` all disappear. Fewer branches, one VDOT source.
- **The funnel narrows to runners with a real race history.** The true never-timed beginner has no on-ramp at MVP — an accepted trade-off given the race/marathon wedge. The 5K test used to be that beginner's baseline path; we are choosing the simpler invariant over the broader on-ramp.
- **`setVdotFromRaceResult` survives** as the post-onboarding way to refresh fitness (the `manual-pace-sheet` on the Fitness page). Only its test-completion *caller* is removed.
- **The change-goal wizard drops its recent-race step** and reuses the athlete's persisted VDOT — VDOT is an athlete-level metric, established once at onboarding, so re-entering a time on every goal change is needless friction.
- **No Agoge change.** The `"test"` workout type remains in the component's enum (and in the shared `WorkoutType` / `WORKOUT_TYPES_COLORS` that mirror it), simply unused by Cadence. It is **hidden from the calendar legend** the same way `recovery` already is (a `CalendarLegendSheet` filter), rather than removed from the type — so a legacy completed test still renders, just without a legend entry.
